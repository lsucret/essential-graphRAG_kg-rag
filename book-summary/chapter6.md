Chapter 6. Constructing knowledge graphs with LLMs

정보 추출 후 구조화  
- 과거엔 어려웠으나 이젠 LLM이 해줌
- openAI는 API에 Structured Outputs 기능도 도입함.(https://developers.openai.com/api/docs/guides/structured-outputs)
출력 형식을 미리 정의, 모델의 응답이 특정 구조를 따르도록 함

ex) "date 속성은 str 타입의 yyyy-mm-dd 형식으로 추출해라"

구조화된 출력을 추출하려면  
- 1단계. 추출할 모델 정의. (with 도메인 전문가)
- 2단계. 구조화된 결과물 추출 요청하는 프롬프트 작성
- 3단계. 데이터셋

출력 후엔 Neo4j에 저장. (neo4j 그래프 모델 설계 필요)  
- 노드: contract, organization, location
- 관계: has_party, has_location, ..

요약  
- 검색을 위해 문서를 단순히 청크로 나누기만 하면 부정확하거나 뒤섞인 결과가 나올 수 있으며, 특히 문서 경계가 중요한 법률 문서 같은 도메인에서는 이런 문제가 더 두드러집니다.
- 필터링, 정렬, 집계와 같은 검색 작업에는 구조화된 데이터가 필요하며, 텍스트 임베딩만으로는 이러한 연산을 적절히 처리하기 어렵습니다.
- LLM은 비정형 텍스트에서 구조화된 데이터를 효과적으로 추출할 수 있으며, 이를 표나 키-값 쌍과 같은 활용 가능한 형식으로 변환할 수 있습니다.
- LLM의 Structured Outputs 기능을 사용하면 개발자가 스키마를 정의할 수 있어, 응답이 특정 형식을 따르도록 보장하고 후처리 필요성을 줄일 수 있습니다.
- 계약 유형, 당사자, 날짜와 같은 속성을 포함한 명확한 데이터 모델을 정의하는 것은, LLM이 관련 정보를 정확하게 추출하도록 안내하는 데 필수적입니다.
- 지식 그래프에서 엔터티 해결은 동일한 엔터티의 서로 다른 표현을 병합하는 데 중요하며, 이를 통해 데이터의 일관성과 정확성을 높일 수 있습니다.
- 지식 그래프에서 구조화된 데이터와 비정형 데이터를 함께 결합하면 원본 자료의 풍부한 정보를 보존하면서도 더 정밀한 질의를 수행할 수 있습니다.


소고  
- 결국 LLM은 추출을 도와줄 뿐 스키마는 내가 만들어야 한다는 거구나
