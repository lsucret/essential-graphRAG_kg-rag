# Chapter 5 agentic RAG

## agentic RAG란?
사용자를 넘어서서 작업을 수행하는 시스템

리트리버 라우터
- 사용자 질문을 입력받아, 어떤 리트리버를 사용하는 것이 적절한지 반환하는 기능
- 
리트리버 에이전트
- 사용자 질문에 답하는 데 필요한 데이터를 검색
- 범용적으로는 벡터 유사도 검색, 구체적으로는 파라미터 입력받아 실행하는 쿼리 템플릿
- text2cyper도 리트리버 에이전트
- 사용자 기대 수준에 맞는 성능을 내려면 특정 목적에 뛰어난 성능을 내는 특화 리트리버가 필요.
응답 평가기(Answer critic)
- 리트리버들이 가져온 답변을 입력받아, 원래 질문에 제대로 답했는지 확인하는 기능
- 차단 역할 + 올바른 답을 검색하는데 사용할 새로운 질문 생성 + 올바른 답이 존재하지 않을 경우를 대비한 종료 조건.

## 5.2 왜 필요한가?
역사적 흐름
- LLM: LLM이 학습한 지식만으로 답변. 최신 정보 반영이 어렵다는 한계.
- 전통적 RAG (naive RAG)
	- 답하기 전에 외부 문서를 검색해서 답변. 모델 내부 지식과 외부 문서 저장소를 결합.
	- 질문 입력 -> 검색기에서 관련 문서 top-k 추출 -> 그 문서를 넣고 답 생성
	- 검색이 한 번으로 끝난다고 가정하는 한계(고정형 파이프라인)
	- 검색 필요여부를 판단하지 못하고, 단순 문서 검색으로 끝나는 한계
- agentic RAG
	- 어떻게 찾을지, 더 찾을지, 찾은게 맞는지를 검증함.

요새는
- "RAG가 더 agentic해지는 흐름"과 "RAG 없이도 더 잘하게 만드는 흐름"이 같이 발전중.
- long-context(리트리버가 답변 조각이 아닌 긴 컨텍스트를 들고옴). structured retrieval(graphRAG 활용), memory(LLM이 검색과정을 기억), policy learning(계획과 검색을 더 잘하게 정책 학습), security/evaluation(보안)이 결합된 더 신뢰할 수 있는 agentic system 쪽으로 가는 중

기타 연구
- test-tims scaling: 답변 전에 얼마나 더 탐색하고 검토할지

검색 결과를 내부화할 순 없을까?
- 세 갈래 연구가 있음
	- continual learning, lifelong learning
	- knowledge editing / lifelong knowledge editing
	- self-adapting / self-updating models
- 가능은 하지만 과거 기억이 지워지거나, 연결이 완전하지 않거나(다른 맥락으로 질문시 학습한 외부정보를 답하지 못하거나), 잘못된 답변을 학습할 수 있음


## 5.3 어떻게 적용하나?

### 5.3.1 리트리버
영화 제목으로 영화를 가져오는 Cypher 템플릿 기반 리트리버 하나, 
배우 이름으로 영화를 가져오는 Cypher 템플릿 기반 리트리버 하나, 
그리고 그 외의 모든 질문을 처리하는 text2cypher 기반 리트리버 하나를 사용할 것

LLM은 어떤 리트리버를 사용할지 선택할 뿐 실제 호출은 시스템이 수행해야 한다.

### 5.3.2 리트리버 라우터 
지속적인 쿼리 업데이팅
- "가상 많은 오스카상을 받은 이는 누구고, 현재 살아 있나요?"
- 첫번째 질문의 답을 얻고 그 정보를 반영해 두번째 질문을 업데이트 해야 한다.
1. handle-tool-calls 함수: 리트리버를 호출
2. query-updater 함수: 쿼리 업데이트
3. route question 함수: 질문을 적절한 리트리버에 연결. 내부에서 1번 함수 호출
4. handle-user-input 함수: 사용자 질문 1개를 받아서, 질문을 보정하고, 적절한 리트리버를 실행한 뒤, 그 결과를 대화 이력 형태로 누적하는 함수

### 5.3.3 응답 평가기
critique-answers 함수: 보완할 질문이 있으면 질문 배열을 리턴

### 5.3.4 전체 실행
main prompt: 유저 질문과 answer 값을 가지고 답변 완성. 답변이 덜 된 부분은 덜 됐다고 말하기. 외부 정보는 사용하지 않기.

main 함수: 리트리버로 답변 조회 -> 응답 평가 1회 -> 답변 생성

### 요약

- agentic RAG는 사용자 질문에 답하는 데 필요한 데이터를 가져오기 위해 여러 검색 에이전트를 사용할 수 있는 시스템입니다.
- agentic RAG 시스템의 주요 인터페이스는 보통 유스케이스 라우터 또는 리트리버 라우터이며, 현재 작업에 가장 적합한 리트리버 하나 또는 여러 개를 찾는 역할을 합니다.
- agentic RAG 시스템의 기본 구성요소는 리트리버 에이전트, 리트리버 라우터, 응답 평가기입니다.
- agentic RAG 시스템의 주요 부분은 tools/function-calling을 지원하는 LLM을 사용해 구현할 수 있습니다.
- 리트리버 에이전트는 범용적일 수도 있고 특화되어 있을 수도 있으며, 애플리케이션 성능 향상을 위해 필요에 따라 점진적으로 추가해 나가야 합니다.
- 응답 평가기는 리트리버들이 가져온 답변들을 받아, 원래 질문에 제대로 답했는지를 확인하는 기능입니다.
